centos与本地上传下载文件--lrzsz

发表于 2017-01-15 | 分类于 原创 |

lrzsz是一款在linux里可代替ftp上传和下载的程序。

首先安装lrzsz

  1. 在线安装

    yum -y install lrzsz

  2. 离线安装
    1.1 到网上下载lrzsz安装包，这里以lrzsz-0.12.20.tar.gz为例
    1.2 打开终端 cd 到安装包所在目录
    tar zxvf lrzsz-0.12.20.tar.gz 解压安装包
    1.3 进入解压好的目录里
    cd lrzsz-0.12.20
    1.4 配置安装路径
    ./configure –prefix=/usr/local/lrzsz
    1.5 编译
    make
    1.6 安装
    make install
    1.7 配置系统命令

    cd /usr/bin
    ln -s /usr/local/lrzsz/bin/lrz rz
    ln -s /usr/local/lrzsz/bin/lsz sz

如果/usr/bin下已存在，删除
1.8 在SecureCRT中测试是否安装成功 ，
输入 rz 然后按回车如果弹出文件选择框，这样就成功了

rz

运行rz命令后弹出选择文件窗口，找到要上传的文件后确认，文件会被上传到Centos当前目录下

2.下载文件，执行命令sz

sz 文件名

运行 sz命令后 弹出选择保存路径窗口，点击确认后 Centos上的文件会被下载到本地目录下

路径设置 ： 此处我用的是SecureCRT，具体的更改 rz上传/sz下载 的默认的路径的方法：

右键点击当前会话session -> Session Options -> Terminal -> Xmodem/Zmodem ->Directories :

Upload : 你要设置的路径

Download:你要设置的路径

Windows下软件安装神器Chocolatey和Mac下 Homebrew 的介绍和使用

发表于 2016-12-20 | 分类于 转载 |

1. Chocolatey

遇到的问题

windows下要安装很多软件，Git，Node，Ruby，Virtualbox，Vagrant，Atom，Brackets，Chrome… 有些软件是你要直接用的，有些是其它软件所依赖的。安装这些东西，你得先找到它们的官方网站，然后下载适合自己系统的版本，双击安装，Next，Next，下一步，下一步 … 完成！安装完以后，你发现还不能用，因为你不知道这个软件还需要你去安装一个其它的东西。
有些软件是需要翻墙的,比如 chrome.升级时还需要翻墙.不可谓不麻烦.

解决的方法

正确的做法是使用包管理工具（Package Manager）去管理这些软件，这里说的是操作系统级别的包管理工具，也就是管理想要安装到操作系统上的软件的工具。在以后，你还会听到项目级别的包管理，比如前端项目用的 Bower ，还有管理 NodeJS 包的 npm 。

包管理工具，可以让我们用几个简单的命令去安装，升级，还有删除软件，并且它会自动给我们解决软件之间的依赖的问题。

Windows 用户可以使用 Chocolatey 这个包管理工具，Mac 用户可以使用 Homebrew 。
